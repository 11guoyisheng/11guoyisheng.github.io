表格的基本结构：
<table>: 定义表格的容器。
<tr>: 表示表格中的一行。
<th>: 表示表头单元格，通常用于标题。
<td>: 表示表格中的数据单元格。

表格的属性：
border: 设置表格边框的宽度。
cellpadding: 设置单元格内容与单元格边框之间的空白。
cellspacing: 设置单元格之间的空白。
width: 设置表格的宽度。
height: 设置表格的高度。

表格的应用场景：
数据展示: 用于展示数据集，如财务报表、时间表等。
布局: 虽然不推荐，但有时用于页面布局。
比较: 用于比较不同项目的特性或参数

不推荐使用表格做布局：

主要原因包括

语义不明确: 表格的语义是用于展示表格数据，而不是用于布局。使用表格进行布局会导致 HTML 结构不清晰，影响可读性和可维护性。
灵活性差: 表格布局通常是固定的，难以适应不同的屏幕尺寸和设备。响应式设计需要灵活的布局方式，而表格难以实现这一点。

影响性能: 表格在渲染时需要浏览器先计算所有单元格的大小，然后再显示整个表格，这可能会导致页面加载速度变慢，尤其是在复杂布局中。

可访问性问题: 使用表格进行布局可能会对使用屏幕阅读器的用户造成困扰，因为屏幕阅读器会将表格内容解释为数据表，而不是布局。
CSS 提供更好的解决方案: 现代 CSS 提供了更强大和灵活的布局工具，如 Flexbox 和 Grid，可以更好地实现复杂的布局需求。
因此，建议使用 CSS 进行网页布局，以提高网页的可维护性、性能和可访问性。
